# Contributing to LinKod
### Guidelines for Contributors • Version 1.0

Thank you for your interest in contributing to the LinKod Programming Language.
LinKod is a mythic‑symbolic security language built on Elements, Seasons, Glyphs,
Rituals, and the Seal. Contributions help strengthen the language, expand its
ecosystem, and improve its long‑term stability.

---

## 1. Ways You Can Contribute

You may contribute by:

- Reporting bugs  
- Suggesting new features  
- Improving documentation  
- Enhancing the specification  
- Creating example programs  
- Submitting pull requests  
- Participating in discussions  

All contributions are welcome as long as they align with LinKod’s design principles.

---

## 2. Code Standards

To maintain consistency and quality, contributors should follow these standards:

- Write clear, readable code  
- Maintain symbolic and mythic consistency  
- Follow the Element and Season logic model  
- Ensure Rituals and Seals are respected in secure modules  
- Avoid unnecessary complexity  
- Document all new glyphs, rituals, or runtime behaviors  

---

## 3. Repository Structure

spec/               — Formal language specification
docs/               — Documentation and guides
glyphs/             — Glyph definitions
runtime/            — VM and interpreter architecture
modules/            — Language modules
examples/           — Sample LinKod programs
tools/              — Developer tools (future)


---

## 4. Branching Model

LinKod uses a simple branching model:

- `main` — stable, production‑ready  
- `dev` — active development  
- `feature/<name>` — new features  
- `spec/<version>` — specification updates  
- `runtime/<component>` — VM or interpreter work  

Pull requests should target **`dev`**, unless otherwise specified.

---

## 5. Pull Request Requirements

Before submitting a PR:

- Ensure your code builds and runs  
- Include documentation for new features  
- Add examples if relevant  
- Follow LinKod’s symbolic conventions  
- Reference any related issues  
- Keep commits clean and descriptive  

---

---

## 7. Security Considerations

LinKod is a security‑focused language. Contributors must:

- avoid introducing insecure patterns  
- respect Ritual and Seal validation rules  
- ensure glyphs with identity implications behave correctly  
- avoid bypassing authentication or integrity mechanisms  

Security‑related contributions may undergo additional review.

---

## 8. Communication

For questions, discussions, or collaboration:

- Open an issue  
- Join the discussion threads  
- Contact Sebastian directly for enterprise or advanced topics  

---

## 9. Contributor Recognition

All contributors will be acknowledged in:

- the repository  
- the documentation  
- future release notes  

Major contributors may be invited to help shape future versions of LinKod.
