# LinKod Brand Guidelines
### Brand Owner: Eric Rush (Sebastian)

These Brand Guidelines define how the LinKod name, symbols, glyphs, and
visual identity may be used. They protect the aesthetic, symbolic, and
mythic identity of the LinKod programming language.

All branding elements are owned exclusively by:

**Eric Rush (Sebastian)**  
Creator & Architect of LinKod.

---

## 1. Purpose of These Guidelines

The LinKod brand is more than a logo or name — it is a symbolic identity
built around:

- Elements (Flow, Flame, Dusk, Stone)
- Seasons (Spring, Summer, Autumn, Winter)
- Ritual structure
- Glyph meanings
- Seal logic
- Mythic and security-focused philosophy

These guidelines ensure that LinKod’s identity remains consistent,
recognizable, and protected as the project grows.

---

## 2. Official Brand Elements

The following elements are considered part of the official LinKod brand:

### **2.1. The Name**
- “LinKod”
- “LinKod Language”
- “LinKod Runtime”
- “LinKod VM”
- “LinKod Pro”
- “LinKod Enterprise”

### **2.2. The Glyph Emblem**
If LinKod uses a primary glyph emblem, it is protected as part of the brand.

### **2.3. Element Symbols**
Flow, Flame, Dusk, and Stone may have visual symbols. These are official
brand assets.

### **2.4. Seal Symbol**
Any symbol representing the Seal or Ritual verification is part of the
official brand.

### **2.5. Color Palette**
If LinKod uses a defined color palette (e.g., Element colors), it is part of
the brand identity.

### **2.6. Typography**
If LinKod uses a specific typeface or style for its logo or documentation
headers, it is part of the brand.

---

## 3. Allowed Uses (No Permission Required)

You may use LinKod brand elements in the following ways:

- Referencing LinKod in tutorials, articles, videos, or documentation
- Showing compatibility (e.g., “Works with LinKod”)
- Using screenshots of the official documentation
- Using the name “LinKod” in descriptive sentences
- Creating unofficial tools that clearly state they are not official

Examples of allowed descriptive use:

- “An extension for the LinKod language”
- “A tutorial for learning LinKod”
- “A module compatible with LinKod”

These uses must not imply endorsement or official status.

---

## 4. Restricted Uses (Permission Required)

You must obtain written permission from **Eric Rush (Sebastian)** before:

- Using the LinKod name in a product title
- Using LinKod branding in marketing or advertising
- Creating merchandise with LinKod symbols or glyphs
- Using LinKod logos or glyphs in your own branding
- Creating a “LinKod Pro,” “LinKod Enterprise,” or similar product
- Using LinKod colors or glyphs to imply official affiliation
- Creating a logo that resembles the official LinKod emblem

Permission may be granted or denied at the sole discretion of the brand owner.

---

## 5. Forbidden Uses (Never Allowed)

The following uses are strictly prohibited:

- Naming a fork “LinKod” or anything confusingly similar
- Using LinKod branding for competing runtimes or engines
- Creating unofficial “LinKod Pro” or “LinKod Enterprise” versions
- Using LinKod glyphs or symbols in a way that implies official status
- Altering LinKod symbols to create fake “official” variants
- Using LinKod branding to mislead users
- Using LinKod brand elements in harmful, malicious, or deceptive contexts

These restrictions protect the integrity of the LinKod identity.

---

## 6. Fork Branding Rules

Forks of LinKod must:

- use a different name
- use different glyphs or symbols
- use different colors
- avoid using LinKod’s emblem or Seal symbol
- clearly state they are unofficial

Examples of acceptable fork names:

- “StoneScript”
- “GlyphForge”
- “FlowLang”
- “Sebastian-Fork-LK” (clearly unofficial)

Forks may not use LinKod branding in any way.

---

## 7. Visual Identity Rules

### **7.1. Colors**
If LinKod uses Element-based colors (e.g., blue for Flow, red for Flame),
these colors are part of the brand and may not be used in unofficial logos.

### **7.2. Glyph Style**
LinKod glyphs have a distinct symbolic style. Unofficial projects may not:

- copy the glyph style
- remix glyphs to imply official status
- create derivative glyphs that resemble official ones

### **7.3. Seal Symbol**
The Seal symbol represents authentication and identity. It may not be used
outside official LinKod documentation or runtime.

### **7.4. Logo Placement**
The LinKod logo may not be placed:

- on unofficial documentation
- on unofficial modules
- on forks
- on commercial products
- on merchandise

---

## 8. Documentation Style

LinKod documentation uses a symbolic, mythic tone. Unofficial projects may
not imitate this tone in a way that implies official status.

Examples of prohibited imitation:

- “Official LinKod Ritual Guide”
- “LinKod Seal Authority Manual”
- “LinKod Elemental Specification”

Unofficial documentation must clearly state it is community-created.

---

## 9. Enforcement

Violations of these guidelines may result in:

- requests to remove branding
- requests to rename projects
- revocation of contribution rights
- public clarification of unofficial status
- legal action in severe cases

Enforcement is at the sole discretion of the brand owner.
